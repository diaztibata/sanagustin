Aplicación Individual - Reglas de Presentación

La aplicación individual consta de 10 actividades, cada una enfocada en desarrollar características esenciales en la creación de software. Cada actividad tiene un valor de 0.5 puntos, por lo que completar 10 actividades resultará en una calificación de 5.0 puntos al final del segundo corte.

La aplicación debe estar instalada en un telefono android y debe estar realizada en web para llamarse a través de webview. Es fundamental que la aplicación utilice la API asignada en clase, según la lista dentro del documento. Solo se podrá cambiar de API si no ha sido asignada a otro compañero, excepto PokeAPI, que está reservada para el docente.
.
Para la entrega final, la aplicación debe cumplir con:
  1. ✅ Contar con al menos 6 pestañas. (0.5 puntos)
  2. ✅ Tener un menú inferior con 4 ítems. (0.5 puntos)
  3. ✅ Tener un buscador. (0.5 puntos)
  4. ✅ Implementar un filtro. (0.5 puntos)
  5. ✅ Listar elementos obtenidos desde la API asignada. (0.5 puntos)
  6. ✅ Garantizar un diseño homogéneo y una buena experiencia de usuario. (0.5 puntos)
  7. ✅ Incluir un splash screen e ícono personalizado. (0.5 puntos)
  8. ✅ Incluir una funcionalidad original (feature único). (0.5 puntos)
  9. ✅ Formulario de registro (7 campos). (0.5 puntos)
  10. ✅ Implementar un CRUD de favoritos en almacenamiento local. (0.5 puntos)
Con autorización del docente, una característica puede cambiarse por otra.
